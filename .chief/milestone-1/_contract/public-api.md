# Milestone 1 Public API Contract

Strict subset of `.chief/_rules/_contract/public-api.md`. Only these symbols are exported from `rigidjs` at the end of milestone-1.

## Exported Symbols

```ts
// from 'rigidjs'
export { struct }
export type { StructDef, StructFields, NumericType }
```

## Types

```ts
export type NumericType =
  | 'f64' | 'f32'
  | 'u32' | 'u16' | 'u8'
  | 'i32' | 'i16' | 'i8'

export type FieldType = NumericType | StructDef<any>

export type StructFields = Record<string, FieldType>

export interface StructDef<F extends StructFields> {
  readonly sizeof: number
  readonly fields: F
  // Internal: offset table + generated handle constructor.
  // NOT part of the contract — may change without notice.
}
```

## Functions

```ts
export function struct<F extends StructFields>(fields: F): StructDef<F>
```

### Semantics

- Pure. Called once per type definition. Allocates no `ArrayBuffer`.
- `sizeof` equals the sum of field sizes in declaration order.
- Nested `StructDef` contributes `sizeof(nested)` bytes inline.
- Empty field map is **not** allowed — throws at call time.
- Duplicate field names are impossible in TS (object literal), no runtime check required.

## Handle Behavior (Internal, but Contractually Observable via Tests)

The handle class generated by `struct()` is not directly constructible by users in milestone-1 (no public container exists yet). Tests access it via an internal test helper.

Observable guarantees:

1. A handle holds a `DataView` and a numeric base offset.
2. Reading a field returns `number` for numeric types, or a sub-handle for nested struct fields.
3. Writing a numeric field calls the matching `DataView.setXxx(off, v, /*littleEndian*/ true)` at the pre-computed absolute offset.
4. Accessing a nested field does **not** allocate on each access — the same sub-handle instance is reused, with its base offset rebased to the parent's buffer slot.

## Not Yet Exported

`slab`, `vec`, `bump`, iterators, `.drop()`, string types — all deferred to later milestones.
